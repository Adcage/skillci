# SkillCI v0.2.0 版本总结

> **发布日期**：2026-05-31
> **版本定位**：Both Mode, LLM Cache, CI Readiness
> **Git Tag**：[v0.2.0](https://github.com/Adcage/skillci/releases/tag/v0.2.0)

---

## 一、版本概述

v0.2.0 是 SkillCI 的重要功能版本，实现了同时运行本地判断与 LLM 判断的能力，并引入了 LLM Judge 缓存机制，让 SkillCI 更适合进入 CI / PR 工作流。

### 核心目标

1. 实现 `mode=both`，同时运行本地规则检查和 LLM 语义判断
2. 实现 LLM Judge 缓存，避免重复 API 调用
3. 完善 GitHub Actions / CI 使用方式
4. 改进报告输出，使其更适合 PR 检查

### 3. LLM Judge Cache（LLM 判断缓存）

**功能说明**：避免重复调用 API，缓存目录 `.skillci/cache/`，缓存文件为 JSON 格式。

**新增文件**：

| 文件 | 行数 | 说明 |

| [`skillci/cache/__init__.py`](skillci/cache/__init__.py) | L1-L4 | 缓存模块入口 |

| [`skillci/cache/judge_cache.py`](skillci/cache/judge_cache.py) | L1-L34 | 缓存管理类 |

**缓存 key 生成**（`skillci/cache/cache_key.py:10-28`）：
```python
BODY_EXCERPT_MAX_LENGTH = 500

def build_judge_cache_key(
    skill: Skill,
    case: SkillTestCase,
    config: JudgeConfig,
) -> str:
    key_data = {
        "skill_name": skill.name,
        "skill_description": skill.description,
        "skill_body_excerpt": skill.body[:BODY_EXCERPT_MAX_LENGTH] if skill.body else "",
        "case_name": case.name,
        "case_input": case.input,
        "provider": config.provider,
        "base_url": config.base_url,
        "model": config.model,
        "prompt_version": PROMPT_VERSION,
    }
    key_string = json.dumps(key_data, sort_keys=True, ensure_ascii=False)
    return hashlib.sha256(key_string.encode("utf-8")).hexdigest()
```

**缓存管理类**（`skillci/cache/judge_cache.py:8-34`）：
```python
class JudgeCache:
    def __init__(self, cache_dir: Path | None = None):
        self.cache_dir = cache_dir or Path(".skillci/cache")

    def get(self, key: str) -> LLMTriggerResult | None:
        # 读取缓存文件，返回 LLMTriggerResult 或 None

    def set(self, key: str, result: LLMTriggerResult) -> None:
        # 写入缓存文件，包含 cached_at 时间戳
```

**缓存集成**（`skillci/evaluator/llm_trigger_judge.py:35-83`）：
```python
def run_llm_judge(
    skill: Skill,
    case: SkillTestCase,
    thresholds: ThresholdConfig,
    provider_name: str = "openai",
    judge_config: JudgeConfig | None = None,
    use_cache: bool = True,
) -> LLMTriggerResult:
    config = judge_config or JudgeConfig()
    effective_cache = use_cache and config.cache

    if effective_cache:
        cache = _get_cache()
        cache_key = build_judge_cache_key(skill, case, config)
        cached = cache.get(cache_key)
        if cached is not None:
            return cached

    # 调用 provider...

    if effective_cache:
        cache.set(cache_key, final_result)

    return final_result
```

**缓存配置**：
```yaml
# skillci.yaml
judge:
  cache: true  # 启用缓存（默认启用）
```

### 4. --no-cache CLI 参数

**功能说明**：禁用 LLM 缓存，每次都重新调用 API。

**命令**：
