# SkillCI v0.2.0 验收清单

## 版本定位

> 同时运行本地判断与 LLM 判断，并让 SkillCI 更适合进入 CI / PR 工作流。

## 一、新增功能清单

### 1. Both Mode（同时运行本地和 LLM 判断）

**命令**：
```bash
skillci test examples/api-doc-writer --mode both --provider mock
```

**验收点**：
- [ ] 同时输出 "Trigger Check"（本地）和 "LLM Trigger Check"（LLM）
- [ ] 显示 Local Metrics（Precision, Recall, F1, Accuracy）
- [ ] 显示 LLM Metrics（Precision, Recall, F1, Accuracy）
- [ ] 显示 Judge Disagreements 数量
- [ ] 最终显示 Result（passed/failed）

---

### 2. Judge Disagreement（判断差异检测）

**说明**：当本地规则和 LLM 判断不一致时，报告会突出显示。

**验收点**：
- [ ] 如果有 disagreement，显示 "Judge Disagreements" 部分
- [ ] 显示每个 disagreement 的 case name、expected、local、llm、reason
- [ ] 如果没有 disagreement，不显示该部分

**测试方法**：
```bash
# 使用 mock provider 运行，正常情况下应该没有 disagreement
skillci test examples/api-doc-writer --mode both --provider mock
```

---

### 3. LLM Judge Cache（LLM 判断缓存）

**说明**：避免重复调用 API，缓存目录 `.skillci/cache/`

**配置**：
```yaml
# skillci.yaml
judge:
  cache: true  # 启用缓存
```

**验收点**：
```bash
# 第一次运行
skillci test examples/api-doc-writer --mode llm --provider mock

# 第二次运行（应命中缓存，速度更快）
skillci test examples/api-doc-writer --mode llm --provider mock
```
- [ ] 第二次运行结果与第一次一致
- [ ] `.skillci/cache/` 目录下有缓存文件
- [ ] 缓存文件包含 `cached_at` 时间戳

---

### 4. --no-cache CLI 参数

**命令**：
```bash
skillci test examples/api-doc-writer --mode llm --provider mock --no-cache
```

**验收点**：
- [ ] 每次都重新调用 provider，不使用缓存
- [ ] `skillci test --help` 中显示 `--no-cache` 参数说明
- [ ] 参数优先级：CLI `--no-cache` > `skillci.local.yaml` > `skillci.yaml`

## 五、功能变更记录

### 新增

- `mode=both`：同时运行本地触发评估和 LLM 判断
- Judge Disagreement：检测并报告本地/LLM 判断差异
- LLM Judge Cache：避免重复模型调用
- `--no-cache` CLI 参数：禁用缓存
- GitHub Actions 使用指南
- PR 友好的 GitHub 报告格式（`--format github`）

### 改进

- 终端和 Markdown 报告展示本地/LLM 对比
- JSON 报告字段完善
- 测试覆盖完善（167 个测试）

### 未实现（v0.3+）

- 多 Provider（Anthropic / Gemini / Ollama）
- 多次运行一致性（runs: 3）
- HTML Dashboard
- 多 Skill 批量测试
