# SkillCI v0.2.0 验收清单

## 版本定位

> 同时运行本地判断与 LLM 判断，并让 SkillCI 更适合进入 CI / PR 工作流。

---

## 前置准备

### 配置 OPENAI_API_KEY

验收 LLM 相关功能需要真实的 API Key：

```bash
# Windows
set OPENAI_API_KEY=sk-xxx

# Linux/Mac
export OPENAI_API_KEY=sk-xxx
```

或者在 `examples/api-doc-writer/skillci.local.yaml` 中配置：
```yaml
judge:
  api_key: sk-xxx
  base_url: https://api.openai.com/v1
  model: gpt-4o-mini
```

## 一、功能验收

### 1. Local Mode（本地规则检查）

**命令**：
```bash
skillci test examples/api-doc-writer --mode local
```

**验收点**：
- [ ] 输出 "Trigger Check" 部分
- [ ] 显示每个 case 的 expected/actual/score
- [ ] 显示 Local Metrics（Precision, Recall, F1, Accuracy）
- [ ] 最终显示 Result（passed/failed）

---

### 2. LLM Mode（LLM 语义判断）

**命令**：
```bash
skillci test examples/api-doc-writer --mode llm
```

**验收点**：
- [ ] 输出 "LLM Trigger Check" 部分
- [ ] 显示每个 case 的 expected/actual/confidence
- [ ] 显示 LLM Metrics（Precision, Recall, F1, Accuracy）
- [ ] 显示 Average Confidence
- [ ] 最终显示 Result（passed/failed）

---

### 3. Both Mode（同时运行本地和 LLM 判断）

**命令**：
```bash
skillci test examples/api-doc-writer --mode both
```

**验收点**：
- [ ] 同时输出 "Trigger Check"（本地）和 "LLM Trigger Check"（LLM）
- [ ] 显示 Local Metrics
- [ ] 显示 LLM Metrics
- [ ] 显示 Judge Disagreements 数量
- [ ] 最终显示 Result（passed/failed）

---

### 4. Judge Disagreement（判断差异检测）

**说明**：当本地规则和 LLM 判断不一致时，报告会突出显示。

**验收点**：
- [ ] 如果有 disagreement，显示 "Judge Disagreements" 部分
- [ ] 显示每个 disagreement 的 case name、expected、local、llm、reason
- [ ] 如果没有 disagreement，不显示该部分或显示 Count: 0

---

### 5. LLM Judge Cache（LLM 判断缓存）

**说明**：避免重复调用 API，缓存目录 `.skillci/cache/`

**验收步骤**：
```bash
# 第一次运行（调用 API）
skillci test examples/api-doc-writer --mode llm

# 第二次运行（应命中缓存，明显更快）
skillci test examples/api-doc-writer --mode llm
```

**验收点**：
- [ ] 第二次运行速度明显更快
- [ ] 第二次运行结果与第一次一致
- [ ] `.skillci/cache/` 目录下有缓存文件（`.json` 文件）
- [ ] 缓存文件包含 `cached_at` 时间戳

---

### 6. --no-cache CLI 参数

**命令**：
```bash
# 禁用缓存，每次都调用 API
skillci test examples/api-doc-writer --mode llm --no-cache
```

**验收点**：
- [ ] 每次都重新调用 provider（速度与第一次相同）
- [ ] `skillci test --help` 中显示 `--no-cache` 参数说明
- [ ] 参数优先级：CLI `--no-cache` > 配置文件 `judge.cache`

---

### 7. JSON 输出

**命令**：
```bash
skillci test examples/api-doc-writer --mode both --json
```

**验收点**：
- [ ] 输出合法 JSON
- [ ] 包含 `local_results`、`llm_results` 字段
- [ ] 包含 `judge_disagreement_count`、`judge_disagreements` 字段

---

### 8. GitHub 报告格式（PR 友好）

**命令**：
```bash
skillci report examples/api-doc-writer --format github
```

**验收点**：
- [ ] 输出紧凑表格格式
- [ ] 包含 Static Health、Local F1、Judge Disagreements、Regressions、Final Result
- [ ] 可通过 `--output` 写入文件

**预期输出示例**：
```markdown
## SkillCI Check

| Section | Result |
|---|---|
| Static Health | passed |
| Local F1 | 1.00 |
| LLM F1 | 1.00 |
| Judge Disagreements | 0 |
| Regressions | 0 |
| Final Result | **passed** |
```

- [ ] 快照保存到 `.skillci/baselines/` 目录

- [ ] 显示 Total/Unchanged/Regressed/Improved 统计

## 五、功能变更记录

### 新增

- `mode=both`：同时运行本地触发评估和 LLM 判断
- Judge Disagreement：检测并报告本地/LLM 判断差异
- LLM Judge Cache：避免重复模型调用
- `--no-cache` CLI 参数：禁用缓存
- GitHub Actions 使用指南（`docs/github-actions.md`）
- PR 友好的 GitHub 报告格式（`--format github`）
- Prompt Version：缓存失效机制

### 改进

- 终端和 Markdown 报告展示本地/LLM 对比
- JSON 报告字段完善（包含 disagreement 数据）
- 测试覆盖完善（167 个测试）

### 未实现（v0.3+）

- 多 Provider（Anthropic / Gemini / Ollama）
- 多次运行一致性（runs: 3）
- HTML Dashboard
- 多 Skill 批量测试
